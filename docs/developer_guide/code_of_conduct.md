# Code of Conduct

sarpyx project spaces are for focused, respectful technical collaboration.

- Keep discussions professional and relevant to the project.
- Report bugs with enough context to reproduce or investigate them.
- Give review feedback that is specific, actionable, and tied to code or docs.
- Do not share private credentials, proprietary data, or sensitive mission
  artifacts in public issues or discussions.

Maintainers may moderate issues, pull requests, discussions, or comments that
make collaboration unsafe or unproductive.
